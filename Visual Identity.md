# Visual Identity & Aesthetic Guidelines

## 1. Design Philosophy

The design follows a **Strict Swiss/Neo-Brutalist** approach. It prioritizes raw information structure over decoration. The aesthetic is defined by:

- **Objectivity:** Removing emotional design elements (shadows, gradients, rounded corners).
    
- **Structure:** Content is organized by visible grids and rigid lines.
    
- **Contrast:** Extreme scale difference between display text and body text.
    

## 2. Color Palette

The site must remain strictly monochromatic.

- **Background:** Pure White (`#FFFFFF`)
    
- **Foreground (Text/Lines):** Pure Black (`#000000`)
    
- **Accents:** None. No greys are used for text; opacity changes are discouraged. The only "color" comes from loaded images (if any).
    

## 3. Typography

**Font Family:** Neo-Grotesque Sans-Serif (e.g., Helvetica Neue, Arial, Inter, or a raw monospace for code).

### Type Scale

The design relies on an exaggerated scale difference.

|   |   |   |   |   |
|---|---|---|---|---|
|**Role**|**Style**|**Size (Desktop)**|**Weight**|**Text Transform**|
|**Super Display**|Used for purely decorative letters (e.g., "S", "P")|`12rem` - `20rem`|Regular/Medium|Uppercase|
|**Section Header**|Major dividers (e.g., "ABOUT/INFOS")|`3rem` - `4rem`|Regular|Uppercase|
|**Sub-Header**|Column titles (e.g., "CONTACT")|`0.875rem`|Bold|Uppercase|
|**Body Text**|Paragraphs, lists|`0.875rem` - `1rem`|Regular|None|
|**Meta/Nav**|Tiny utilitarian text|`0.75rem`|Regular|Uppercase/None|

### Type Treatment

- **Line Height:** Tight for headers (`0.9` to `1.0`), open for body text (`1.4` to `1.6`).
    
- **Tracking (Letter Spacing):**
    
    - Headers: Tight (`-0.02em`).
        
    - Small Caps/Nav: Loose (`0.05em`).
